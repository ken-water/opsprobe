# OpsProbe v0.12.3

## Summary

`v0.12.3` refines the Windows packaging-chain diagnosis again. The blocker on this machine is now explicitly reduced to a missing current-version Cargo crate cache entry plus failed download fallback, not a vague NSIS or product-side unknown.

## Included Milestones

- `0.12.x` vendoring and cache diagnosis checkpoint

## Key Changes

- identify that the current machine has `linux-raw-sys 0.12.1` cached but not `0.12.2`
- record that `.opsprobe-vendor` remains aligned to `0.12.1`
- document that online recovery attempts are not safe enough to treat as a fix on this machine

## Scope Boundary

Included in `0.12.3`:

- refined diagnosis of the current vendoring/cache blocker
- explicit handoff guidance for the next Windows packaging attempt

Deferred to the next checkpoint:

- hydrating the missing current-version crate through a working machine or network path
- producing the current-version Windows NSIS installer
- real Windows installer acceptance

## Known Limits

- current-version Windows GNU packaging is still blocked on this machine before installer generation begins
- this release improves precision of the blocker diagnosis, not the final Windows acceptance outcome
