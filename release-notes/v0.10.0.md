# OpsProbe v0.10.0

Released: 2026-06-07

`v0.10.0` starts the pre-stable `0.10.x` hardening line with a unified storage boundary for schedules, desktop settings, backup guidance, and machine-move expectations.

## Highlights

- Adds a shared state repository so schedules and desktop settings can live in the same active storage boundary as assets, templates, and inspection runs
- Automatically migrates legacy `desktop-settings.json` and `inspection-schedules.json` files into the active state store
- Extends config export and import to include desktop settings while continuing to force credential rebind for imported assets
- Documents storage layout, backup scope, and machine-replacement guidance for local-first deployments

## Why this release exists

`0.9.4` documented that schedules and desktop settings were still outside the main PostgreSQL-backed runtime path. `0.10.0` closes that first pre-stable credibility gap by unifying active state behavior across PostgreSQL and file fallback modes, then documenting what operators should back up and how they should move between machines.
