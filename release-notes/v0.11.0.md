# OpsProbe v0.11.0

## Summary

`v0.11.0` opens the stable-candidate line by combining packaging hardening, evidence discipline, and a more inspect-first desktop workflow into one traceable checkpoint.

## Included Milestones

- `0.11.x` stable-candidate evidence line

## Key Changes

- establish the `0.11.x` version line for stable-candidate and packaged-desktop work
- harden Linux desktop packaging with vendored build support, cargo cache hydration, and stricter environment gates
- fix bundled local-service runtime resolution for packaged desktop artifacts
- focus the desktop workflow on first inspection, report follow-up, and repair-first system guidance
- add version-specific issue and acceptance documents for the `0.11.0` evidence scope

## Scope Boundary

Included in `0.11.0`:

- Linux-first packaged desktop credibility work
- first-inspection workflow tightening
- stable-candidate evidence structure and issue tracking
- packaging and environment-gate hardening

Deferred to the next version:

- stricter patch-checkpoint release discipline on the active `0.11.x` line
- another pass on report/system workspace simplification
- Windows acceptance and later stable-decision work

## Known Limits

- Windows installer acceptance is still not closed in this checkpoint
- the desktop `System` workspace is clearer than before, but it still needs a lower-noise operator path

## Next Target Version

- `0.11.1`
