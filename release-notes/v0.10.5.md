# OpsProbe v0.10.5

## Summary

`v0.10.5` continues the pre-stable desktop hardening line by simplifying the desktop operator flow and tightening version discipline so in-progress work is always attached to a visible version target.

## Included Milestones

- `0.10.x` pre-stable desktop hardening

## Key Changes

- simplify desktop navigation to `Home`, `Inspect`, `Results`, and `System`
- make the inspection path clearer with target, preview, and automation stages
- stop carrying new work under the previously released `0.10.4` version number
- add a gate that fails when the working version still matches the latest published tag while new commits already exist on top

## Scope Boundary

Included in `0.10.5`:

- navigation and workspace naming simplification
- clearer inspection journey structure
- development-version separation from the latest published release
- release-discipline gate updates for version traceability

Deferred to `0.10.6`:

- another round of UI density and hierarchy tightening inside `Inspect`
- broader consistency work for tables, spacing, and secondary controls
- any extra packaged-release validation needed specifically for the simplified desktop flow

## Known Limits

- `0.10.5` is not published yet
- release artifacts, release-readiness validation, tag, and GitHub release notes still need to be completed before shipping

## Next Target Version

- `0.10.6`
