# OpsProbe v0.11.6

## Summary

`v0.11.6` simplifies the `Reports` and `History` reading path so operators see the current conclusion first, the recommended next actions second, and export/history controls after that.

## Included Milestones

- `0.11.x` desktop operator-flow simplification

## Key Changes

- restructure the report workspace into a clearer top-down reading flow: current conclusion, next actions, export, then trends and history
- bring severity and evidence-count context into the main conclusion area so users do not have to jump across cards to understand result weight
- keep history comparison available, but explicitly secondary to the current result

## Scope Boundary

Included in `0.11.6`:

- report and history workspace hierarchy cleanup
- clearer result-reading flow
- matching UI regression validation

Deferred to the next version:

- additional desktop simplification outside reports/history
- packaged acceptance and stable-decision evidence work

## Known Limits

- `0.11.6` improves readability and export flow, but the overall desktop still needs more refinement before a `1.0.0` stability claim
