# OpsProbe v0.11.8

## Summary

`v0.11.8` continues the desktop simplification line by reducing button competition in `System` and making `Assets` feel more like a single save-and-reuse flow instead of a mixed action page.

## Included Milestones

- `0.11.x` desktop operator-flow simplification

## Key Changes

- reduce the `System` return area to clearer primary choices and a lighter refresh action
- keep `Assets` focused on saving one proven target first, then reusing saved targets
- demote config export/import into a later migration step so it no longer competes with the primary target-saving flow

## Scope Boundary

Included in `0.11.8`:

- system and assets workspace hierarchy cleanup
- button-priority simplification
- matching UI regression validation

Deferred to the next version:

- additional desktop simplification outside system/assets
- packaged acceptance and stable-decision evidence work

## Known Limits

- `0.11.8` improves first-run clarity, but the full desktop still needs more polishing before a `1.0.0` stability claim
