# OpsProbe v0.10.8

## Summary

`v0.10.8` closes the `0.10.x` desktop hardening line by recording Windows validation status, repairing release-history gaps, and tightening release discipline before the stable-candidate evidence line begins.

## Included Milestones

- `0.10.x` pre-stable desktop hardening

## Key Changes

- add a structured Windows installer validation record for the current desktop release line
- add an explicit Windows Wine validation record so unsupported local validation environments are recorded honestly instead of implied
- restore the missing `v0.10.6` release record and tag so the published `0.10.x` history is contiguous again
- tighten release and development-gate rules around contiguous version history, unreleased development versions, and GitHub `Latest` handling after backfilled releases
- prioritize the post-`0.10.8` path around stable-candidate evidence instead of more open-ended desktop polish

## Scope Boundary

Included in `0.10.8`:

- Windows validation recording for the desktop release line
- release-history repair for the missing `0.10.6` publication gap
- release-discipline and gate clarifications for future patch and pre-stable work
- roadmap and issue-line alignment for the next stable-candidate stage

Deferred to `0.11.0`:

- clean-machine and stable-candidate evidence consolidation
- Windows installer acceptance evidence on a real Windows-capable environment
- explicit `1.0.0` blocker review and release-or-defer decision support

## Known Limits

- `0.10.8` records Windows validation state, but it does not yet prove real Windows installer acceptance on a Windows machine
- stable-candidate evidence is still deferred to the `0.11.x` line and is not part of this patch release

## Next Target Version

- `0.11.0`
