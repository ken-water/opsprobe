# OpsProbe v0.7.0

Released: 2026-06-04

`v0.7.0` starts the external-validation cycle with a guided first-run experience. New users can inspect realistic sample runs, review remediation output, and export reports before connecting any real infrastructure.

## Highlights

- Adds desktop onboarding mode persistence with explicit `demo` and `real` paths
- Bundles sample nginx and Kubernetes inspection runs for safe first-run exploration
- Labels sample history clearly so demo content never looks like persisted local-service data
- Updates the desktop release copy to align with the `0.7.0` milestone focus

## Why this release exists

OpsProbe now needs feedback from real operators, not just internal feature accumulation. A guided demo path reduces setup friction and makes it easier for early users to comment on report quality, remediation usefulness, and missing inspection coverage.

## Validation

- `npm run desktop:typecheck`
- `npm --workspace @opsprobe/desktop run build`
- `cargo check --manifest-path apps/desktop/src-tauri/Cargo.toml --config 'source.crates-io.registry="sparse+https://index.crates.io/"'`
