# OpsProbe v0.11.14

## Summary

`v0.11.14` turns the patch-growth concern into an explicit repository rule. OpsProbe now treats `x.y.9` as the default patch ceiling and allows `0.11.x` to continue only inside a documented closeout window through `0.11.16`.

- `0.11.x` release-discipline closeout checkpoint
