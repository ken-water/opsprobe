# OpsProbe v0.10.1

Released: 2026-06-07

`v0.10.1` continues the pre-stable `0.10.x` hardening line by requiring migrated credentials to pass SSH validation before recurring schedules can resume.

## Highlights

- Adds a distinct `verification-required` credential state so rebound credentials are not treated as trusted immediately after editing or import
- Blocks schedule enable and resume behavior until the latest saved asset has passed a successful SSH test
- Persists successful SSH verification back into local storage so later schedule actions can trust the saved asset state
- Automatically imports schedules as disabled with an explicit verification message when the related asset still needs credential validation

## Why this release exists

`0.10.0` unified the storage boundary, but a migrated asset could still look usable before its rebound credential had actually been tested. `0.10.1` closes that gap by making successful SSH validation the condition for recurring schedule trust instead of relying on a filled secret path alone.
