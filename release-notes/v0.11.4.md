# OpsProbe v0.11.4

## Summary

`v0.11.4` continues the desktop simplification line by turning the `Inspect` workspace into a clearer single-path flow with less lateral scanning and stronger step visibility.

## Included Milestones

- `0.11.x` desktop operator-flow simplification

## Key Changes

- replace the previous split inspect layout with a more linear sequence: step strip, target setup, verify-and-preview, then result review
- keep the current connection, template, and check-count context visible through compact summary pills instead of extra narrative blocks
- align unit and browser UI coverage with the updated inspect wording and result heading

## Scope Boundary

Included in `0.11.4`:

- inspect-page hierarchy cleanup
- linear first-run inspection flow improvement
- matching UI regression updates

Deferred to the next version:

- broader desktop polish outside the inspect path
- packaged acceptance and stable-decision evidence work

## Known Limits

- `0.11.4` makes the inspect path easier to follow, but the full desktop experience still needs more simplification before a `1.0.0` stability claim
- this checkpoint improves workflow clarity, not packaging credibility or stable-release evidence

- `0.11.5`
