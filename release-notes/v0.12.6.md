# OpsProbe v0.12.6

## Summary

`v0.12.6` turns the earlier packaging recovery work into a full current-version artifact checkpoint. Linux desktop bundles now build again on this machine, the Windows GNU installer is present for the same version, and the packaged validation evidence has been refreshed so it no longer depends on stale walkthrough records.

## Included Milestones

- `0.12.x` packaged build recovery completion checkpoint

## Key Changes

- repair the vendored `linux-raw-sys 0.12.2` metadata mismatch blocking current package builds
- export `APPIMAGE_EXTRACT_AND_RUN=1` automatically for Tauri's cached Linux bundling AppImage on systems without `libfuse.so.2`
- rebuild and validate current-version Linux `deb`, `rpm`, `AppImage`, and `AppDir` outputs
- rebuild the current-version Windows GNU NSIS installer and refresh Windows validation records
- refresh the desktop operator walkthrough so packaged validation evidence points to `0.12.6` throughout

## Scope Boundary

Included in `0.12.6`:

- restored current-version desktop package generation on this machine
- current-version packaged evidence for Linux, Windows artifact output, and walkthrough coverage

Deferred to the next checkpoint:

- native Windows installer acceptance on a real Windows runtime
- final pre-stable release-hardening work beyond package generation recovery

## Known Limits

- Windows validation is still based on the GNU cross-build output from Linux plus recorded evidence, not a native MSVC build
- native Windows install-and-run proof is still a follow-up item before any stable release decision
