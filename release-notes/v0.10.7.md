# OpsProbe v0.10.7

## Summary

`v0.10.7` continues the pre-stable desktop hardening line by making first launch clearer, reducing workflow clutter, and attaching packaged desktop validation evidence to the current version instead of older bundle output.

## Included Milestones

- `0.10.x` pre-stable desktop hardening

## Key Changes

- reshape the desktop start page into an install-first launch surface with clearer next actions and less duplicated explanation
- tighten the inspection workflow into a more direct target -> SSH -> preview -> result path
- refocus the report strategy workspace around audience choice, remediation framing, and missing-feedback intake
- add packaged validation recording so the current version can be checked against matching `deb`, `rpm`, `AppImage`, preflight, launch-smoke, and walkthrough evidence
- build and validate `0.10.7` Linux desktop bundles for `deb`, `rpm`, and `AppImage`

## Scope Boundary

Included in `0.10.7`:

- desktop first-launch and start-page clarity
- straighter first inspection execution flow
- report strategy and feedback-space cleanup
- current-version packaged desktop validation evidence on Linux

Deferred to the next version:

- Windows installer validation for the same release line
- GitHub release publication polish and the next development-version bump
- broader stable-candidate work beyond the current `0.10.x` desktop hardening scope

## Known Limits

- `0.10.7` packaged validation is Linux-first; Windows installer evidence is still missing
- packaged validation proves the current version bundles and launches, but it does not yet replace broader real-user acceptance coverage
