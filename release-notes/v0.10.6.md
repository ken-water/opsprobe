# OpsProbe v0.10.6

## Summary

`v0.10.6` continues the pre-stable desktop hardening line by tightening the simplified desktop workflow, reducing secondary clutter, and adding broader browser-level regression coverage for the main operator path.

## Included Milestones

- `0.10.x` pre-stable desktop hardening

## Key Changes

- streamline the desktop inspection flow so first-run operators move through the main path with less ambiguity
- reduce setup and report clutter so secondary explanation no longer competes with the main actions
- refocus the results workspace so the current conclusion and export actions appear before history detail
- tighten overall desktop UI density across cards, spacing, and interaction hierarchy
- align browser-level desktop flow coverage with the simplified workspace structure

## Scope Boundary

Included in `0.10.6`:

- denser `Inspect` workflow hierarchy
- reduced `Setup` and `Reports` clutter
- clearer `Results` workspace priority
- desktop UI density and consistency polish
- updated browser regression coverage for the simplified operator path

Deferred to `0.10.7`:

- install-first start-page feedback and clearer first-launch guidance
- broader packaged desktop validation tied to the current version
- any additional release evidence needed for the next stable-candidate checkpoint

## Known Limits

- `0.10.6` improves the day-to-day desktop flow, but packaged-install and first-launch credibility were still not strong enough to resume the stable decision
- Windows installer and packaged acceptance evidence were still missing in this release line

## Next Target Version

- `0.10.7`
