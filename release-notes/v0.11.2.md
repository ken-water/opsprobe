# OpsProbe v0.11.2

## Summary

`v0.11.2` simplifies the desktop `System` workspace into a blocker-first repair page so operators can understand what prevents the first real inspection and where to go next without sorting through reference-heavy UI.

## Included Milestones

- `0.11.x` desktop operator-flow simplification

## Key Changes

- reduce `System` to a blocker-first readiness summary, focused repair packs, and one explicit return path
- remove the old reference-heavy troubleshooting surface and duplicated demo-mode guidance
- align browser UI coverage with the new `System` information architecture
- clean up stale desktop helper code left behind by the refactor

## Scope Boundary

Included in `0.11.2`:

- `System` workspace UX simplification
- duplicate-guidance cleanup
- matching UI regression updates

Deferred to the next version:

- additional stable-candidate evidence review and release-or-defer work
- any future multilingual, website, or collaboration features

## Known Limits

- `0.11.2` makes `System` much easier to scan, but the overall product still needs more work before a `1.0.0` stability claim
- packaged acceptance and stable-decision evidence remain a separate track from this UX checkpoint

- `0.11.3`
