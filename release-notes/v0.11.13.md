# OpsProbe v0.11.13

## Summary

`v0.11.13` tightens desktop action hierarchy so the main operator path is easier to scan: run first, then review, then export, with destructive runtime actions clearly separated.

## Included Milestones

- `0.11.x` desktop workflow simplification checkpoint

## Key Changes

- move `History` topbar actions to a simpler run-and-refresh model and keep export/open-file actions inside the active result card
- split `Service` controls into health, OpsProbe service, and bundled PostgreSQL groups with explicit danger styling for stop/delete paths
- keep `Assets` focused on saving and reusing targets while demoting transfer import/export into migration utilities
- align browser regression coverage to the new export action labels

## Scope Boundary

Included in `0.11.13`:

- desktop action grouping and hierarchy cleanup
- danger-button treatment for destructive desktop actions
- regression updates for renamed report file actions

Deferred to the next version:

- deeper visual density cleanup across detail cards
- broader navigation simplification beyond the current workspace action surfaces

## Known Limits

- this release improves action clarity, but the desktop information density still needs another pass in later `0.11.x` patches
