# OpsProbe v0.3.0

## Summary

`v0.3.0` establishes the dedicated local service foundation behind the desktop application and moves runtime ownership out of the Tauri UI process.

## Included Milestones

- Local service runtime boundary
- Managed local PostgreSQL lifecycle and health checks
- Service-owned inspection preview and execution
- PostgreSQL-preferred inspection persistence

## Key Changes

- Added a dedicated `local-service` package and command boundary queried from the desktop UI
- Added service status inspection and local service lifecycle controls in the desktop application
- Added managed PostgreSQL prerequisite checks, bootstrap, start, stop, and runtime health reporting
- Added service-owned inspection preview and execution flow over SSH
- Added PostgreSQL-backed inspection run persistence with file-backed fallback and automatic migration of legacy file-backed runs
- Hardened local PostgreSQL shutdown and storage cleanup behavior

## Known Limits

- Filterable inspection history was not included yet
- Local recurring scheduling was not included yet
- Machine migration import/export was not included yet
- First-run setup guidance was not included yet
- Local HTML and PDF report export were still deferred

## Next Target

- `0.4.0` Local usability with history, scheduling, migration support, and onboarding guidance
