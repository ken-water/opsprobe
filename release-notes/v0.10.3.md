# OpsProbe v0.10.3

Released: 2026-06-08

`v0.10.3` completes the pre-stable `0.10.x` hardening line by making local runtime recovery and machine replacement behavior more explicit, reviewable, and operator-guided.

## Highlights

- Adds structured runtime `recoveryActions` and a dedicated `restart` flow so operators can recover from stale runtime markers without guessing
- Surfaces restart and recovery guidance directly in the desktop local-service panel
- Records source-machine provenance in exported config packages and reports post-import rebind and disabled-schedule counts before automation resumes
- Extends functional, migration, and smoke validation to cover restart preparation and machine-move trust signals

## Why this release exists

`0.10.0` through `0.10.2` hardened the storage boundary, credential verification, and recovery evidence, but the local runtime story still relied too much on operator inference. `0.10.3` closes that gap by making restart guidance, post-failure recovery, and machine-move trust signals explicit in both product behavior and release validation.
