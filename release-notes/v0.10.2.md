# OpsProbe v0.10.2

Released: 2026-06-08

`v0.10.2` continues the pre-stable `0.10.x` hardening line by proving that corrupted local state and older export packages can recover cleanly instead of forcing manual repair.

## Highlights

- Quarantines malformed file-backed storage snapshots into timestamped `.corrupt-*` files and rebuilds a clean state file automatically
- Quarantines malformed local-service status snapshots and avoids stale PID files reporting a false `ready` state
- Extends migration compatibility coverage for older export packages that do not include desktop settings
- Expands smoke validation to exercise malformed storage recovery, malformed status fallback, stale PID fallback, and post-recovery asset saves

## Why this release exists

`0.10.0` unified the storage boundary and `0.10.1` required real SSH verification before schedules could resume, but recovery evidence was still too thin. `0.10.2` closes that gap by hardening the common failure paths around corrupted local snapshots, stale runtime metadata, and older config packages.
