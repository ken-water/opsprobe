# OpsProbe v0.7.1

Released: 2026-06-05

`v0.7.1` reduces early-user setup drop-off by turning common SSH, local-service, and managed PostgreSQL failures into readable troubleshooting guidance.

## Highlights

- Adds a desktop troubleshooting guidance panel for failing environment checks
- Converts common SSH failures into clearer authentication, network, DNS, and key-permission explanations
- Surfaces actionable repair suggestions for missing binaries, report directory permissions, and managed PostgreSQL port conflicts
- Improves command failure visibility for local service health, schedules, assets, and inspection history operations

## Why this release exists

Early users should not need to inspect source code or raw logs just to understand why setup failed. This release makes common environment problems visible inside the product so external feedback can stay focused on real needs instead of basic setup friction.

- `npm run desktop:typecheck`
- `npm --workspace @opsprobe/desktop run build`
