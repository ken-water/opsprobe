# OpsProbe v0.12.0

## Summary

`v0.12.0` starts the final bounded pre-stable hardening line. This release does not reopen broad product work. It prepares the exact evidence records needed to close the last Linux and Windows packaged acceptance gaps before revisiting `1.0.0`.

## Included Milestones

- `0.12.0` final pre-stable packaged acceptance hardening

## Key Changes

- start the `0.12.0` line after the `0.11.16` stable defer decision
- add current-version Linux packaged acceptance and Windows installer acceptance records
- link the `0.12.0` issue scope directly to those records so later proof is captured against the active version

## Scope Boundary

Included in `0.12.0`:

- version-line start for the final packaged acceptance stage
- current-version acceptance record templates
- issue-scope alignment for the last pre-stable hardening line

Deferred to the next checkpoint:

- actual Linux packaged human acceptance evidence
- actual Windows installer acceptance evidence
- reopened `1.0.0` decision after those proofs exist

## Known Limits

- `0.12.0` only starts the final hardening line; it does not yet close the remaining packaged acceptance blockers
- no new feature expansion is included in this line
