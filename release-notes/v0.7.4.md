# OpsProbe v0.7.4

Released: 2026-06-05

`v0.7.4` closes the external-validation cycle with an explicit stabilization checkpoint and a documented decision to continue into `0.8.0` exploration rather than declaring `1.0.0`.

## Highlights

- Adds a desktop validation summary panel for the full `0.7.x` cycle
- Adds a `0.7.x` decision document covering addressed friction, deferred scope, and next-stage rationale
- Updates repository docs so the current product-stage decision is easy to find

## Why this release exists

The goal of the `0.7.x` line was not just to ship isolated features. It was to learn whether first-run comprehension, setup friction, report fit, and feedback routing were becoming clearer. This release records that learning explicitly and uses it to choose the next stage with evidence rather than assumption.

## Validation

- `npm run desktop:typecheck`
- `npm --workspace @opsprobe/desktop run build`
- `cargo check --manifest-path apps/desktop/src-tauri/Cargo.toml --config 'source.crates-io.registry="sparse+https://index.crates.io/"'`
