# OpsProbe v0.11.11

## Summary

`v0.11.11` improves consistency across table-based workspaces by introducing a shared list header and aligning how list pages present counts, purpose, empty states, and loading states.

## Included Milestones

- `0.11.x` desktop operator-flow simplification

## Key Changes

- add a shared list-header shell for reusable table surfaces
- use the shared list framing across `Assets`, `History`, and `Schedules`
- remove duplicate count emphasis where the list shell already provides that context

## Scope Boundary

Included in `0.11.11`:

- reusable table/list-surface consistency improvements
- shared empty/loading/list framing
- matching UI regression validation

Deferred to the next version:

- additional desktop simplification outside list-based surfaces
- packaged acceptance and stable-decision evidence work

## Known Limits

- `0.11.11` improves consistency, but the overall desktop still needs more refinement before a `1.0.0` stability claim

- `0.11.12`
