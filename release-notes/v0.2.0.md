# OpsProbe v0.2.0

## Summary

`v0.2.0` delivers the first end-to-end Linux SSH inspection flow for OpsProbe.

This release introduces the shared inspection domain, the SSH connection test path, and a baseline set of Linux host checks executed through the desktop runtime.

## Included Milestone

- `0.2.0` First Inspection Flow

## Key Changes

- Added shared domain models, check contracts, and runner adapter flow
- Added an editable Linux host asset form in the desktop UI
- Added SSH connection testing through the Tauri backend
- Added SSH-backed Linux baseline checks:
  - CPU usage
  - Memory usage
  - Disk usage
  - Load average
  - Time synchronization
  - sshd process status
  - TCP port 22 listening state
  - Last reboot time
  - `/var/log` usage
- Added version development gate checks for issue and milestone hygiene

## Known Limits

- The desktop edition still runs without the planned local service process
- The managed PostgreSQL runtime is not implemented yet
- Linux checks currently focus on a baseline host set and do not yet cover Docker, Kubernetes, MySQL, Redis, or Nginx
