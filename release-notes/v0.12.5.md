# OpsProbe v0.12.5

## Summary

`v0.12.5` turns the current packaged-build blocker into a concrete offline recovery workflow. Instead of only documenting that `linux-raw-sys-0.12.2.crate` is missing on this machine, OpsProbe now supports preloading missing `.crate` files from another machine before vendor refresh is attempted.

## Included Milestones

- `0.12.x` offline vendoring recovery checkpoint

## Key Changes

- add `OPSPROBE_PRELOADED_CRATE_DIRS` support to `scripts/hydrate-cargo-cache.sh`
- copy matching preloaded `.crate` files into the local Cargo cache before online download fallback
- document the preload workflow in the Linux packaged-evidence runbook and environment preparation help text

## Scope Boundary

Included in `0.12.5`:

- a deterministic offline recovery path for missing vendored crates
- updated operator documentation for current packaging recovery

Deferred to the next checkpoint:

- obtaining `linux-raw-sys-0.12.2.crate` from another machine or working network path
- refreshing `.opsprobe-vendor` with that crate
- rerunning Linux packaged build validation
- rerunning Windows installer validation

## Known Limits

- this release adds the recovery path but does not yet include the missing crate payload
- packaged artifact generation is still blocked on this machine until the preload step is actually performed
