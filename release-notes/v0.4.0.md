# OpsProbe v0.4.0

## Summary

`v0.4.0` turns OpsProbe from a single inspection shell into a local-first desktop workflow with a dedicated local service, persistent history, recurring schedules, machine migration support, and first-run environment guidance.

## Included Milestones

- Local service runtime foundation
- Managed local PostgreSQL lifecycle and storage preference
- Inspection history and replay
- Local recurring scheduling
- Configuration export and import for machine migration
- First-run setup guidance and environment validation

## Key Changes

- Added a desktop-to-local-service runtime boundary for status, preview, execution, scheduling, migration, and history
- Added managed PostgreSQL bootstrap, start, stop, health inspection, and file-to-PostgreSQL run migration
- Added filterable inspection history with reopened run details and repeated-problem summaries
- Added local recurring inspection schedules stored and executed by the background service
- Added local config export/import with secret-safe credential rebinding on the destination machine
- Added first-run setup guidance covering local service startup, saved assets, credential rebinding, and report directory validation
- Added a strict checkpoint gate so issue work is committed and pushed before continuing

## Known Limits

- Local HTML report export is not released yet
- Local PDF export is not released yet
- Asset persistence and broader restart polish remain tracked separately under issue `#12`
- Managed PostgreSQL binaries still need to be available on the local machine in the current open source flow

## Next Target

- `0.5.0` Open Source MVP polish with result view model, local HTML/PDF reporting, and remaining persistence cleanup
