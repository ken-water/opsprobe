# OpsProbe v0.11.3

## Summary

`v0.11.3` continues the desktop simplification line by making the `Start`, `Inspect`, and `Reports` workspaces more obvious at first glance and less dependent on explanatory copy.

## Included Milestones

- `0.11.x` desktop operator-flow simplification

## Key Changes

- simplify `Start` into a clearer first action, readiness check, and post-preview follow-up view
- tighten the `Inspect` entry into a `Now` versus `Later` split so manual inspection stays visually primary
- remove duplicated audience explanation from `Reports` and keep one cleaner report-mode selection surface
- align desktop unit and browser UI coverage with the revised wording and task hierarchy

## Scope Boundary

Included in `0.11.3`:

- first-run workspace hierarchy cleanup
- report-mode duplication reduction
- matching UI regression updates

Deferred to the next version:

- deeper packaged acceptance and stable-decision evidence work
- future multilingual, website, and collaboration features

## Known Limits

- `0.11.3` makes the first-run path easier to scan, but the overall desktop product still needs more end-to-end refinement before a `1.0.0` stability claim
- stable-candidate evidence and release-or-defer review remain separate from this UX checkpoint

## Next Target Version

- `0.11.4`
